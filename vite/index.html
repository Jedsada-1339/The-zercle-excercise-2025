A. What is the role of the index.html file?

The index.html file serves as the entry point for a web application. It loads the base HTML structure and links to other resources like CSS and JavaScript files.

B. What is the URL you used to view the page? Is it HTTP or HTTPS?

The URL used is usually something like http://localhost:5173, which uses the HTTP protocol (not HTTPS).

C. What HTTP status code do you see for index.html? What does it mean?

The status code is typically 200, which means "OK" â€” the request has succeeded and the file was delivered.

D. How can you tell the page is being served locally vs from a real domain?

If the address bar shows localhost or an IP like 127.0.0.1, then it's being served locally. A real domain would look like www.example.com.

E. What tools did you use to inspect the response?

Browser Developer Tools (F12), especially the "Network" tab, are used to inspect the response status, headers, and content.
